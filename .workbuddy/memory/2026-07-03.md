## 墨白视觉设计稿完成（2026-07-03）

### 交付物
- templates/design-mockup.html - 完整视觉设计稿（带 mock 数据，可直接预览）
- templates/design-notes.md - 设计决策说明文档

### 设计要点
1. **配色系统**：四态三层（填充/边框/文字 + glow），暗色模式非简单反色而是重调对比度
2. **布局**：Phase 分组 + 卡片网格（非树状连线，依赖用 chip 标签表达）
3. **动效**：6 种（状态脉冲/进行中旋转/报错抖动/hover 磁吸/Phase 入场/进度弹性）
4. **字体**：等宽字体只用于 ID/文件路径/commit hash
5. **性能**：所有动效只用 transform 和 opacity，50 任务不卡

### 给吴八哥的数据接口
任务对象需新增字段：fileStats（每个文件是否存在）、completedDate、errorMessage
详见 design-notes.md 第六节

### v3 视觉改版（用户第二次反馈后重做）
用户反馈：第二版还是"动态施工图"，老板看不懂依赖关系，要看"装修进度图"
重做方向（老板视角）：
- 砍掉所有节点和连线（那是工程师看的）
- 主体是每个 Phase 一个横向进度条卡片，分段式（每段=1个任务）
- 顶部大进度环 + 四个关键指标卡（完成/进行中/报错/未开始）
- 报错阶段整张卡片红色脉冲边框，最醒目
- 进行中任务段有流光动画（shimmer）
- 状态标签用文字：已完成/进行中/卡住了/未开始（不用 T001 这种 ID）
- 文件：templates/design-mockup.html（覆盖了 v2）

关键认知：用户是老板视角不是工程师视角，要的是"全局进度感"不是"技术拓扑"
